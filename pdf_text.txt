Introduction  
  
This comprehensive guide provides all the information you need to seamlessly integrate Version 
3 of our powerful translation plugin into your website, enabling smooth translation between 
English and Bhartiya languages.   
  
Key Features:  
  
1. Seamless Translation : Effortlessly translate your website content between English and 
Bhartiya languages with high accuracy.   
2. Glossary Addition : Utilise glossary for domain specific translations, enabling accurate and 
context aware translations.   
3. Selective Translati on: Skip translation for specific text enclosed within certain tags, ensuring 
precise control over what gets translated.   
4. Customizable Placement:  Position the translation plugin widget on your website's bottom 
left or bottom right, tailored to your design  preferences.  
5. Preferred Language Storage:  Remember the language used for translating a page in a 
website and auto -translate any page from the website to the previously translated language 
when opened in a browser.  
  
By following this guide, you'll be abl e to leverage the full potential of the BHASHINI Translation 
Plugin, making your website more accessible and user -friendly for a multilingual audience.

Translation Plugin Implementation Steps  
  
The Bhashini Plugin can be embedded in any website using the following approach. The 
integration behaviour may differ based on the framework being used.   
 
For Vanilla JS / Static HTML Sites  
Step 1: Add the plugin script at the bottom of the <body> tag in index.html  
<script src="https://translation -
plugin.bhashini.co.in/v 3/website_translation_ utility.js"></script>   
Properties of the Script Tag  
1. src :  
○ Definition : The src  attribute specifies the URL of an external script file to be 
executed.  
○ Explanation : In this case,  
https://translation -plugin.bhashini.co.in/v 3/website_t  
ranslation_utility.js  is the URL that points to the JavaScript file for 
the BHASHINI Translation Plugin. This file contains the necessary code to enable 
translation functionality on yo ur website.  
 
Step 2: Wherever you want the plugin to appear, add a container with the class bhashini -
plugin -container:  
 
<div class="bhashini -plugin-container"></div>  
 
 
                       
  
4  
Since vanilla JS sites load and render DOM synchronously, the plugin script will immediately find 
the target container and mount itself without issues.  
 
 
 
 
For React / Next.js or other SPA frameworks  
 
Add  useEffect inside the last component, such  as Footer  
 
useEffect(() => {  
     const script = document.createElement("script");  
     script.src = "  https://translation -plugin.bhashini.co.in/v3/website_t  
ranslation_utility.js ";  
     script.async = true;  
    document.body.appendChild(script);  
  }, []);  
 
This ensures the plugin is injected only after  the component is mounted and the target DOM 
element exists.  
 
 
2. In Next.js, place the script inside useEffect or load it using next/script with 
strategy="afterInteractive"  to avoid hydration issues

Additional Features  
  
Skip Translation:   
By adding the class bhashini -skip-translation  to any HTML element, you can  
exclude that specific element from being translated by the BHASHINI Translation Plugin. This is 
useful for text that should remain in its original language, such as b rand names, technical terms, 
or any other content you want to keep unchanged.  
Below is an example that demonstrates how to use the BHASHINI Translation Plugin on your 
webpage and how to skip the translation of certain text by using the class “bhashini -skip-
translation”.  
  
<body>   
<!-- Your website content goes here -->   
<!-- BHASHINI Translation Plugin Script -->   
 <p> Translate This </p>         
<p class="bhashini -skip-translation" >Skip Translating This</p>  
<script src="https://translation -
plugin.bhashini.co.in/v 3/website_transla 
tion_utility.js"></script> </script>   
</body>

Preferred Language Storage:  
  
Whenever a user utilizes your website and translates a specific page to an Indic language 
and moves to a different webpage in your site, the new webpage gets translated to the  
previously translated Indic language by keeping a track of preferred language storage 
system in the browser’s local storage.  
  
FAQs  
  
1. How do I display Manipuri (Meetei) Font if it’s not installed on the user’s computer?   
To ensure the Manipuri (Meetei) font is displayed correctly even if it’s not installed on the user's 
computer, you can import the necessary font from Google Fonts. Add the following line of code 
to your CSS file:  
@import  
url('https://fonts.googleapis.com/css2?family=Noto+Sans+Meetei+Mayek:wght@100..90  
0&display=swap');  
This will load the "Noto Sans Meetei Mayek" font, which supports the Manipuri (Meetei) script, 
ensuring it displays correctly on your website regardless of the  fonts installed on the user's 
device.  

                       
13  
As a fallback option, you can specify alternative fonts in case the primary font fails to load.  
Here's an example of how to set up your CSS to include a fallback: 
body {   font-family: ‘Arial', 'sans -serif','Noto Sa ns Meetei 
Mayek', ;  
}  
In this example, if the "Arial" font or a generic sans -serif font cannot be loaded, the browser will 
attempt to use "Noto Sans Meetei Mayek" font as a backup. This ensures that text is always 
displayed, even if the preferred font is n't available.  
  
2. Is it possible to skip translation for any specific tags?   
Yes, as specified in the Implementation Steps section, you can skip tags to be translated by using 
“bhashini -skip-translation” class for the specific tag.   
  
3. I have the plugin retranslating the page in certain circumstances. For example, when a 
dropdown menu is hovered on, it retranslates that content, even though it doesn’t change. 
How do I resolve this?  
This issue occurs because the plugin detects a DOM reload, where the text content is internally 
replaced even if it hasn't changed visibly. When the DOM is reloaded, the plugin treats it as new 
content and retranslates it.  
To avoid this, you should minimize D OM reloads by ensuring that the content is managed in a 
way that doesn’t trigger these reloads. One effective approach is to style the content using inline 
CSS. By doing so, you reduce the chances of the DOM being reloaded when interacting with 
elements li ke dropdown menus, which in turn prevents unnecessary retranslations.

3. Certain keywords are translated wrongly. How do I resolve this?  
BHASHINI offers Translation Plugin services with Glossary System to override default translations 
with domain specific translations.  
Example:  
● Default Translation: "Digital India Bhashini  Division" is translated to " डिजटल  इं िया भै 
सनी प्रभाग ".  
● With Glossary Term: If the glossary term between English and Hindi is entered as 
"डिजटल  इं िया भा षणी िवीज़न ", this will override the default translation.  
To utilize glossary within your system, please reach out to your onboarding manager.
