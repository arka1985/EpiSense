🔍 Why Combine Tests?

No single screening test is perfect. Healthcare professionals often combine tests to maximize the chances of correctly identifying a disease. Here are real-world examples:

  • Parallel Testing (Net Sensitivity): Multiple tests run at once. Example: An ER doctor ordering multiple blood cultures and imaging at the same time for suspected sepsis because missing the infection is fatal. Net sensitivity increases, specificity decreases.
  • Sequential Testing (Net Specificity): Tests run one after another. Example: Standard HIV screening where a highly sensitive ELISA test is done first, and if positive, a highly specific Western Blot is done to confirm. Net specificity increases, sensitivity decreases.

📈 Concept of ROC Curve Analysis

The Receiver Operating Characteristic (ROC) curve is a graphical plot that illustrates the diagnostic ability of a binary classifier system as its discrimination threshold is varied.

It is created by plotting the True Positive Rate (Sensitivity) on the Y-axis against the False Positive Rate (1 - Specificity) on the X-axis at various threshold settings.

  • AUC (Area Under the Curve): Measures the total two-dimensional area underneath the ROC curve. A model whose predictions are 100% wrong has an AUC of 0.0; one whose predictions are perfectly correct has an AUC of 1.0.
  • Perfect Test: Passes through the upper left corner (100% Sensitivity, 0% False Positives).
  • Useless Test: Follows the diagonal line (AUC = 0.5), meaning it is no better than randomly flipping a coin.
